接口

上面了解到了抽象类的局限性，解决这个问题最理想的方式就是使用接口

1. 什么是接口？

Java中接口的作用和生活中的接口类似，它提供一种约定，使得实现接口的类（或结构）在形式上保持一致
概念性的接口，指系统对外提供的所有服务
interface 定义的是实实在在的接口，即接口类型
接口的存在弥补了Java单根继承的缺点

JDK8之前的接口特性

1.接口中的变量都是静态常量( public static final ) ，必须显式初始化
2.接口中所有方法默认都是public abstract
3.接口没有构造方法，不可以被实例化，但可以被实现(常作为类型使用)
4.实现类必须实现接口的所有方法
5.实现类可以实现多个接口 (Java中的多继承)
6.implements、多个接口使用逗号隔开

USB接口可以被多种USB设备实现
